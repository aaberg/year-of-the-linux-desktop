# Year of the Linux Desktop?

Suggested outline for a 15-minute conference talk.

## Core Thesis

2026 may not be “the year Linux wins the desktop” for everyone, but for developers, gamers, privacy-conscious users, and people tired of vendor-controlled PCs, Linux is now good enough, often better, and increasingly attractive.

## 1. Opening: The Joke That Refuses To Die

Time: 1 minute

Start with the classic phrase:

> “Every year is the year of the Linux desktop. Except it never is.”

Then reframe the question:

> “But maybe we have been asking the wrong question. Not: has Linux conquered the desktop? But: has Windows made Linux the better default for more people?”

## 2. The Modern Windows Experience

Time: 3 minutes

Describe what happens when you buy a new Windows PC in 2026:

- Candy Crush
- Ad-supported Solitaire
- Microsoft Teams
- Clipchamp
- OneDrive nudges
- Bing and Edge pushing
- OEM tools like Lenovo Vantage, Asus bloatware, and Acer bloatware
- Apps you never asked for

Main point:

> “A clean Windows install is no longer clean. It is a negotiation.”

Mention that people now use tools like `Win11Debloat`, with tens of thousands of GitHub stars, just to get back to a sane starting point.

## 3. Telemetry Is Not Just Privacy, It Is Performance

Time: 2 minutes

Move from annoyance to cost.

Windows collects and processes data such as:

- Hardware specs
- App usage
- Error reports
- Network data
- Optional data like browsing history, typing patterns, and location

The important argument:

> “Telemetry is not only a privacy issue. It is also a performance issue.”

Use benchmark notes as supporting examples:

- Clean boot: Ubuntu around 8 seconds, Windows 11 around 18 seconds
- Idle RAM: Ubuntu around 1.2 GB, Windows 11 around 3.5 GB

## 4. The Trust Problem: Recall, Copilot, TPM

Time: 2 minutes

Frame this section as a loss of ownership.

Topics:

- Windows 11 requires TPM 2.0
- Recall takes screenshots of user activity
- Copilot is integrated throughout the system
- Features may be “off”, but the software is still there
- User choice feels increasingly secondary

Key line:

> “The problem is not only what Windows does today. It is that the direction is clear: more cloud, more AI, more telemetry, more control moving away from the user.”

## 5. Linux Has Quietly Become Practical

Time: 3 minutes

This is the optimistic middle section.

Concrete improvements and signals:

- Steam Deck shipped in 2022
- Proton started in 2018 and has matured significantly
- Linux is around 5% in the Steam survey, according to the notes
- NTSYNC improves performance for some Windows games
- Bazzite installation took around 15 minutes and included Steam and Nvidia drivers
- Framework sells Ubuntu machines, and reportedly more Ubuntu than Windows on some models

Key line:

> “Linux did not become successful by becoming Windows. It became successful by solving enough real-world problems that people stopped caring that it was Linux.”

## 6. Why Linux Feels Better

Time: 2 minutes

Make this section personal and experiential.

Themes:

- Linux does what you ask it to do
- No forced Microsoft account narrative
- No default ad ecosystem
- No pushing Edge, Bing, OneDrive, or Copilot into every corner
- Real customization
- Search and file operations feel fast
- The system feels like yours

Useful contrast:

> “On Windows, I spend time removing things. On Linux, I spend time choosing things.”

## 7. The Developer Superpower: Fixability

Time: 1 minute

Use the developer angle:

> “If something does not work on Linux, I can often inspect it, script it, patch it, or ask OpenCode or Claude Code to help fix it. Try doing that with Windows internals or macOS system behavior. Good luck.”

Main point:

> “Linux is not always easier. But it is more fixable.”

## 8. Closing: So, Is It The Year?

Time: 1 minute

Avoid claiming total victory.

> “Is 2026 the year Linux dominates the desktop? Probably not.”

Then make the stronger point:

> “But it may be the year where Linux stops being the weird choice, and Windows becomes the choice you have to justify.”

Possible final line:

> “The year of the Linux desktop is not when everyone switches. It is when ordinary technical users look at Windows, look at Linux, and honestly ask: why am I still putting up with this?”

## Suggested Slide Flow

1. Title: `Year of the Linux Desktop?`
2. The old joke
3. Buying a Windows PC in 2026
4. Debloating as a mainstream activity
5. Telemetry: privacy plus performance
6. Recall, Copilot, TPM: who owns the machine?
7. Benchmarks: boot, RAM, install experience
8. Gaming changed everything: Steam Deck, Proton, NTSYNC
9. Linux hardware is real: Framework, Bazzite, Ubuntu
10. Linux feels like ownership
11. Developers can fix things again
12. Conclusion: not domination, but legitimacy

## Tone Recommendation

Use a slightly provocative but not fanboy-ish tone. The strongest version of this talk is not “Windows bad, Linux perfect.” It is:

> “Windows has become hostile enough, and Linux has become practical enough, that the old joke deserves another look.”
